# Lago - README Reference

**Source:** github.com/getlago/lago | **Stars:** 12,000+ | **License:** AGPL v3

## Key Patterns to Learn From

- **Competitive positioning:** "The best alternative to Chargebee, Recurly or Stripe Billing"
- **Feature list organized by use case** (metering, pricing, coupons, add-ons, invoicing, prepaid credits)
- **Docker-first deployment** — `docker compose up` gets you running
- **Three stated principles:** hybrid pricing, composable architecture, transparent pricing
- **Submodule architecture** — separate repos for API, frontend, and client libraries
- **Community links** — Slack, docs, GitHub discussions, public roadmap

## Original Content Summary

Lago is an open-source metering and usage-based billing API. Supports event-driven usage tracking, flexible pricing plans (pay-as-you-go + hybrid), coupons, add-ons (one-time charges), automatic invoicing, and prepaid credits. Uses Docker/Docker Compose. Has client libraries for Go, JavaScript, Python, Ruby. AGPL licensed with emphasis on no vendor lock-in and no percentage-of-revenue pricing.

## What Token Wallet Can Copy

- Competitive positioning works if done tastefully ("Unlike X, we focus on Y")
- Docker quickstart for self-hosted deployments
- State your principles — it attracts aligned contributors
- Multiple client library SDKs show maturity
- Public roadmap link builds trust and community engagement
